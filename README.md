## 一、基础召回链路设计

### 1.1 多路召回架构原理

#### 核心概念
- **定义**：在召回阶段并行启用多条异构策略，每条策略基于不同特征或算法，各自从全库中截取少量候选，再通过动态配额算法融合成最终候选送入排序层
- **业务价值**：
  - 提升天花板：单路召回率约40%，五路互补后可达65%以上
  - 改善长尾与冷启动：图谱路与向量路对长尾商品覆盖率提升10%
  - 平衡效率与效果：总延迟控制在15ms内

#### 常见召回通道对比

| 召回通道 | 适用场景 | 优势 | 劣势 |
|---------|---------|------|------|
| Item-CF行为协同 | 淘宝详情页"看了又看" | 对新用户无需画像，10ms级返回 | 对冷启动商品效果差 |
| 双塔语义向量 | 抖音短视频推荐 | 兼顾语义泛化与实时性 | 需要大量训练数据 |
| 实时Session图召回 | 拼多多直播间"换一换" | 分钟级捕捉热点 | 计算复杂度高 |

### 1.2 倒排索引构建与更新

#### 物理存储结构
```
┌─────────────────────────────────────────────────────────┐
│                    倒排索引结构                          │
├─────────────────────────────────────────────────────────┤
│  TermDict段 (FST)                                       │
│  ├── 热词常驻 HugePage (2MB大页)                        │
│  └── 冷词按 LRU 换出                                    │
├─────────────────────────────────────────────────────────┤
│  Postings段                                             │
│  ├── Block: 64KB                                        │
│  ├── Chunk: 4KB                                         │
│  └── SubChunk: 256 DocID                                │
└─────────────────────────────────────────────────────────┘
```

#### 延迟优化
- 热词FST常驻HugePage（2MB大页），冷词按LRU换出
- 超高频词（DF>500万）转RoaringBitmap，Seek操作变位运算
- 单次倒排召回1000个DocID的P99延迟约0.7ms

### 1.3 文本语义召回技术

#### BM25打分公式
```
Score(Q, D) = Σ_{t∈Q} IDF(t) · [tf(t,D)·(k1+1)] / [tf(t,D)+k1·(1-b+b·|D|/avgdl)]
```

**参数物理意义**：
- `k1`：词频饱和拐点，电商标题通常取1.2~2.0
- `b`：文档长度惩罚系数，商品搜索一般取0.75
- `IDF(t)`：抑制高频词、放大长尾词

#### 双塔DSSM负样本构造策略

| 策略 | 优点 | 缺点 |
|-----|------|------|
| 随机负采样 | 实现极简，训练吞吐高 | 与线上分布差异大 |
| Batch内负采样 | 显存零增加，负例与正例同分布 | 热门Item被过度惩罚 |
| 曝光未点击负采样 | 标注置信高，与业务指标耦合强 | 存在位置偏差 |
| 难负例挖掘 | 样本难度平滑可控 | 计算链路重，运维成本高 |

## 三、多任务排序学习

### 3.1 共享底层网络设计

#### MMoE、PLE、CGC对比

| 结构 | 参数隔离度 | 门控稳定性 | 线上效果 |
|-----|-----------|-----------|---------|
| MMoE | 0%（全部Expert共享） | 易出现"赢者通吃" | CVR涨1%时CTR掉0.6% |
| CGC | ≈30%（仅底层隔离） | 中等 | 中等 |
| PLE | >60%（每层共享/独占并存） | 门控熵始终>0.8 | CTR+1.2%、CVR+2.1%、GMV+3.4% |

### 3.2 跷跷板现象缓解策略

#### 根本原因
1. **目标函数梯度冲突**：CTR与CVR在共享Embedding上的梯度夹角>90°
2. **样本空间不一致**：高CTR样本与高CVR样本重叠率<20%
3. **资源硬约束**：首屏曝光位有限，任何目标增益都挤占其他目标流量

#### 可观测指标
- CTR-CVR联合分布的Pearson相关系数由正转负
- GMV/曝光与CTR在在线分层实验中7日滑动斜率反向变化
- Pareto超体积（PV）连续3天下跌超过2%

## 四、重排与多样性策略

### 4.1 行列式点过程（DPP）原理

#### 核矩阵构造
```
L_ij = q_i · q_j · S_ij
```
其中：
- `q_i`：商品i的质量分
- `S_ij`：商品i与j的相似度

#### 多样性强度调节
- 通过温度系数τ调节：`L_ij = q_i · q_j · S_ij^τ`
- τ越大，多样性越强

### 4.2 强化学习重排模型

#### MDP建模
- **状态**：用户历史行为序列、当前候选集、上下文特征
- **动作**：对候选集的排列顺序
- **奖励**：List-wise奖励函数（点击、转化、停留时长加权）
