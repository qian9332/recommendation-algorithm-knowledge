## 💼 推荐后端工程师岗位全景

### 核心定位

```
构建高可用、高性能、可扩展的推荐系统基础设施
```

### 日常工作清单

```
【每日必做】
├── 09:00-09:30 查看系统监控（P99延迟、QPS、错误率）
├── 09:30-10:00 处理告警和工单
├── 10:00-12:00 核心开发工作
├── 14:00-17:00 系统优化和维护
└── 17:00-18:00 文档和总结

【每周必做】
├── 周会：同步项目进度、风险
├── 技术分享：团队内部技术交流
├── 代码评审：参与团队代码评审
└── 容量评估：评估系统容量

【每月必做】
├── 系统巡检：全面检查系统健康度
├── 成本优化：评估资源使用效率
└── 故障复盘：总结本月故障和改进
```

### 核心技术栈

| 类别 | 技术 |
|-----|------|
| **编程语言** | Java、C++、Python |
| **分布式系统** | 微服务、消息队列、缓存、数据库 |
| **推荐特有** | 向量检索、特征存储、模型服务、实时计算 |
| **监控运维** | Prometheus、Grafana、ELK、K8s |

### 核心系统模块

```
用户请求 → 网关层 → 召回服务 → 粗排服务 → 精排服务 → 重排服务 → 响应组装

支撑服务：特征服务 | 模型服务 | 配置服务 | 监控服务
```

### 工作优先级

| 优先级 | 类型 | 示例 |
|--------|------|------|
| P0 | 立即处理 | 线上故障、服务不可用 |
| P1 | 当天处理 | 性能严重下降、核心功能异常 |
| P2 | 本周处理 | 性能优化、代码重构 |
| P3 | 排期处理 | 体验优化、工具开发 |
