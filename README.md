## 🔑 核心知识点速查

### 一、基础召回链路设计

#### 多路召回架构
- **定义**：并行启用多条异构策略，各自从全库截取候选，融合后送入排序层
- **业务价值**：召回率从40%提升到65%+
- **常见通道**：Item-CF、双塔向量、知识图谱、实时行为序列

#### 倒排索引优化
- 热词FST常驻HugePage，冷词LRU换出
- 超高频词转RoaringBitmap
- P99延迟<0.7ms

#### 双塔DSSM负样本构造
| 策略 | 效果 |
|-----|------|
| 随机负采样 | 基准 |
| Batch内负采样 | +1.5% AUC |
| 曝光未点击 | +2% AUC |
| 难负例挖掘 | +3% AUC |

### 四、重排与多样性策略

#### DPP核矩阵
```
L_ij = q_i · q_j · S_ij
```
- q_i：质量分
- S_ij：相似度
- 温度系数τ调节多样性强度

#### 100ms内Top-50采样
- 核矩阵稀疏化：复杂度O(N³)→O(NK²)
- 特征降维：768维→32维
- 并行计算：AVX512加速
- 总耗时：29~35ms

#### 疲劳衰减函数
| 类型 | CTR提升 | P99延迟 |
|-----|--------|---------|
| 指数衰减 | +1.17% | +2.1ms |
| 阶跃衰减 | +1.21% | +0.8ms |
| 自适应衰减 | +1.86% | +4.6ms |
