# 六、影响模型效果与性能的关键细节

> 本章汇总PDF中那些非常细节但极其重要的知识点，这些细节往往决定了模型效果的成败。

## 6.1 偏差纠正框架对比

### DCR、DCN、ESM² 适用条件

| 框架 | 适用场景 | 纠偏类型 | 工程代价 |
|-----|---------|---------|---------|
| **DCR** | 广告精排 | 选择偏差 | 需可靠倾向得分模型，延迟+5% |
| **DCN** | 电商重排 | 位置偏差 | 多塔部署，延迟+2ms |
| **ESM²** | 成交导向场景 | 样本选择偏差+转化稀疏 | 三塔结构，延迟+10~15ms |

### 选择建议

```
1. 若线上已有广告真实p(click|show)日志，且延迟预算<5ms
   → 优先选DCR，双稳健估计可保证方差可控

2. 若业务痛点是Top-N位置偏差导致头部商品过度集中
   → 选DCN，用bias塔学偏差，线上主塔减bias塔即可

3. 若目标是GMV且转化样本稀疏(<0.1%)
   → 用ESM²在全空间直接建模曝光未点击样本的counterfactual转化概率
```

## 6.2 重要性采样方差上界

### 核心公式推导

```
IPS估计量方差上界：

Var[ȳ_IS] ≤ R_max / n

其中：
- R_max = max_x π(x)/π₀(x)  (最大权重比)
- n = 样本量

关键洞察：
当π与π₀分布差异大（策略探索剧烈）时，R_max可达10²~10³
导致离线评估置信区间极宽
```

### 降方差技巧

| 技术 | 效果 | 实现复杂度 |
|-----|------|-----------|
| 权重截断(clipping) | 直接限制R_max | 低 |
| SNIPS | 归一化权重 | 低 |
| Doubly Robust | 结合模型预测 | 中 |
| PDR | 更稳定的DR变体 | 中 |

## 6.3 多任务模型离线AUC提升但在线负向的原因

### 五大根因

```
1. 样本与分布漂移
   └── 离线训练用"有偏曝光样本"
   └── 线上面对"全流量无偏分布"
   └── 多任务共享层过度拟合高频组合

2. 任务间梯度冲突（跷跷板效应）
   └── 共享层被强任务（CTR）主导
   └── 弱任务（CVR）梯度被淹没
   └── 排序公式pCTR×pCVR×bid中CVR偏低

3. 位置与上下文偏差放大
   └── 多任务模型对position bias敏感
   └── 模型把"位置信号"误学为"兴趣信号"
   └── 头部越推越热、腰部彻底沉底

4. 打分校准失效
   └── 离线AUC只关心相对序
   └── 不检查概率值是否校准
   └── 融合打分分布发生整体漂移

5. 系统延迟与特征穿越
   └── 实时用户序列特征P99延迟120ms
   └── 打分阶段取到部分空值
   └── 模型退化为"低阶特征"模式
```

### 解决方案

```
上线前必做：
1. 分布一致性校验
2. 任务梯度可视化
3. 概率校准（Platt Scaling / Isotonic Regression）
4. 延迟压测
5. 长期收益仿真
```

## 6.4 DPP核矩阵构造与多样性强度调节

### 核矩阵公式

```
L_ij = q_i · q_j · S_ij

其中：
- q_i：商品i的质量分（CTR、GMV、毛利率等）
- S_ij：商品i与j的相似度（余弦相似度）
```

### 多样性强度调节方法

| 方法 | 公式 | 特点 |
|-----|------|------|
| **全局温度T** | L(T) = q^{1/T} q^{1/T T} ⊙ S | T→0纯贪心，T→∞只考虑多样性 |
| **可学习对角增广** | L_α = L + diag(α) | α_i越大越容易被抑制 |
| **分层强度** | 按场景分桶，每桶独立T和α | 场景级多样性控制 |

### 在线回退机制

```
当实验桶GMV跌幅>2%时：
├── 自动把T置0.1
├── α置0
└── 秒级回退到纯贪心，保障收入
```

## 6.5 100ms内完成Top-50 DPP采样的优化

### 核心优化策略

```
1. 核矩阵稀疏化
   └── 相似度截断Top-K（K=50）
   └── L变成分块对角+低秩
   └── 采样复杂度从O(N³)降到O(NK²)

2. 特征降维
   └── 768维BERT向量 → OPQ16压缩到64维 → PCA到32维
   └── 计算det时精度损失<1%
   └── 耗时降8×

3. 并行计算
   └── AVX512 + FMA一次算16对φ_iφ_j
   └── CPU周期降4×
   └── OpenMP按128条batch分片

4. 早停策略
   └── 当剩余候选最大可能Δ_i小于已选末位增益1%时截断
   └── 实测可在k=40处停，节省20%计算
```

### 耗时预算

```
50轮贪心 × (0.35ms并行求Δ + 0.08ms更新) ≈ 21.5ms
特征加载 + 核构造 12ms
早停截断节省 4ms
总耗时 29~35ms，P99 <45ms
```

## 6.6 强化学习重排的方差控制

### REINFORCE vs Actor-Critic vs DDPG

| 算法 | 梯度标准差/均值 | 所需session量 | 在线更新延迟 |
|-----|----------------|--------------|-------------|
| REINFORCE | 8.3 | 2000万 | >15分钟 |
| Actor-Critic | 2.1 | 450万 | 3分钟 |
| DDPG | 2.5 | 580万 | 4分钟 |

### 方差来源分析

```
1. 轨迹级蒙特卡洛采样
   └── 单条session的回报R(τ)对策略θ的梯度估计仅依赖一条样本
   └── 零均值噪声大

2. 动作空间维度诅咒
   └── K越大，logπ(a|s)的协方差矩阵秩不足
   └── 高维重要性权重方差爆炸

3. 奖励稀疏
   └── 有效正样本<1%
   └── 正负梯度更新比例失衡
```

### 推荐方案

```
Actor-Critic是当前国内重排场景的最优折中：
├── 通过Critic网络把延迟奖励分解为即时优势
├── 方差下降4倍
├── 配合离线预训练+多样性正则
└── 成为美团、抖音、淘宝2023新架构主流方案
```

## 6.7 用户疲劳衰减系数量化

### 疲劳数学定义

```
同一类目第k次曝光带来的边际效用ΔU随k增加而下降：
ΔU_k = ΔU_0 · e^(–λk)

λ即为疲劳衰减系数
```

### 三种疲劳函数对比

| 函数类型 | 公式 | CTR提升 | GMV提升 | P99延迟增加 |
|---------|------|--------|--------|------------|
| 指数衰减 | w(k)=exp(−λk) | +1.17% | +0.94% | +2.1ms |
| 阶跃衰减 | w(k)=β if k≤T else α | +1.21% | +1.05% | +0.8ms |
| 自适应衰减 | w(k)=1/(1+γ·pCTR·k) | +1.86% | +1.63% | +4.6ms |

### 落地建议

```
1. 团队人力紧张、缓存层改造空间有限
   → 优先阶跃衰减，收益确定且代码改动<30行

2. 多样性是年度OKR，且具备实时特征平台
   → 投入自适应疲劳，γ更新收口到自动化实验平台

3. 指数衰减
   → 可作为baseline写进A/B框架模板
```

## 6.8 重排阶段疲劳因子引入（不增加P99延迟）

### 零拷贝拼接方案

```
1. 离线预计算疲劳分数
   └── 天级Spark任务把"用户×候选商品"的疲劳分预计算成0/1/2三档
   └── 压缩成2 bit，共4档
   └── 灌入Redis的Fatigue-Hash

2. 近线增量更新
   └── 用户每次曝光后，Flink CEP作业在200ms内写回Redis
   └── Hash过期滑动6h

3. 在线零拷贝拼接
   └── 重排服务通过Redis Pipeline一次性批量(mget 200个key)
   └── 把2-bit疲劳档直接映射到预置的3个浮值嵌入向量
   └── 用位运算常量时间完成
   └── 嵌入向量与原模型concat后喂入最后一层LR
   └── 耗时增加<0.3ms
```

### 延迟兜底

```
若Redis超时≥2ms或返回空：
├── 自动降级为0档
├── 保证无阻塞
└── 压测结果：P99从28.4ms→28.7ms，涨幅<1ms
```

## 6.9 CLV（长期用户价值）多任务评估框架

### CLV定义

```
CLV = ∑(t=1→24) E[边际利润_t × 存活概率_t × 折现因子_t]

国内电商一般取24个月
边际利润不含拉新券，避免高估
```

### 多任务拆解

```
主任务：p(CLV|user,context)
辅助任务：
├── p(留存天数)
├── p(高客单概率)
└── p(复购周期)

共享底层MMoE，用不确定性加权动态调节loss
```

### 评估体系

| 维度 | 指标 |
|-----|------|
| 离线-回归 | MAE/MAPE按十分位桶看系统性高估/低估 |
| 离线-排序 | NDCG@CLV，看模型能否把高CLV用户排到前面 |
| 离线-因果 | Uplift-Qini，对比"推模型"vs"不推模型"的CLV增量 |
| 在线-短期 | 7天GMV、CTR、转化率不掉线 |
| 在线-中期 | 28天留存率、复购率 |
| 在线-长期 | 分群CLV、Payback Period（回收周期） |

## 6.10 Off-policy修正（IPS+DR）降低重排策略评估方差

### Doubly Robust估计公式

```
V̂_DR(π₁) = 1/n ∑ᵢ [ ρᵢ·(r₀−q̂(a₀,x)) + ∑_{a∼π₁(·|x)} q̂(a,x) ]

其中：
- ρᵢ = π₁(a₀|x) / π₀(a₀|x)  (级联IPS权重)
- q̂(a,x) 为奖励预测器

三级裁剪降低方差：
1. 权重裁剪：ρ̃ = min(max(ρ, 0.01), 20)
2. 分位点自适应：小时级实时流计算ρ的95%分位
3. 奖励塑形：残差(r₀−q̂)做Winsorize
```

### 实测效果

```
抖音电商重排AB实验：
├── IPS相对标准差：28%
├── DR相对标准差：9%
├── DR偏差：<0.5%
└── 支撑新策略全量上线，单日GMV +3.1%
```

## 6.11 冷启动商品探索不足的缓解方案

### 四层落地框架

```
第一层：不确定性导向的探索奖励
├── bonus = α · σ(s,a)，σ为Dropout不确定性
├── α按商品上架时长指数衰减
└── 保证"越新越敢探"

第二层：元知识初始化（Meta-initialization）
├── 离线训练商品侧元网络
├── 输入商品类目、价格带、主图向量、标题BERT向量
├── 输出SAC策略网络的初始权重θ₀和初始Q值
└── 采用MAML框架，目标"在新商品上仅需10次曝光即可超越随机策略"

第三层：安全保量策略（Safe Exploration with Budget）
├── 业务侧给冷启商品分配日级探索预算B=总流量×5%
├── 按PID控制器动态调整
└── 若实时监测到GMV损失率>1.5%，立即收缩预算

第四层：在线仿真回放（Simulator-based Warm-up）
├── 构建新品仿真器
├── 用GAN+用户序列模拟器生成"伪曝光→伪点击→伪下单"数据
├── 首日即可为冷启商品生成10万条虚拟样本
└── 冷启商品上线前先在仿真器里预训练1000迭代
```

### 实测效果

```
抖音电商2023年双11：
└── 新品首小时成交速度提升42%
```

## 6.12 List-wise奖励函数构造

### 会话定义

```
以"用户一次下拉刷新到下次下拉刷新"为一次推荐列表
含K=10条卡片
```

### 单卡片即时奖励

```
r_i = 0.4·normalize(停留时长_i) + 0.3·完播_i + 0.8·成交_i − 0.6·负反馈_i

其中 normalize(x) = min(x, 30s) / 30s
```

### List-wise会话奖励

```
R = Σ_{i=1}^{K} γ^{i−1} · r_i − λ·快滑率

γ=0.95 做位置衰减
λ=0.5 惩罚快滑率>0.4的会话
```

### 偏差修正

```
对position 1~3的样本乘IPS权重：
w_i = 1 / P(停留|pos=i)

其中P用随机沉底曝光样本离线拟合
```
