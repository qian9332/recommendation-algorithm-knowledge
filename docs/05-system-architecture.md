## 5.4 模型监控与可观测

### 黄金四指标

| 指标 | 定义 | 告警阈值 |
|-----|------|---------|
| 延迟 | P99响应时间 | >30ms |
| QPS | 每秒请求数 | 下降>10% |
| 错误率 | 请求失败比例 | >0.1% |
| 饱和度 | CPU/GPU利用率 | >85% |

### 模型效果漂移监控

```
实时监控指标：

1. Calibration Error
   └── 预测CTR与实际CTR的累积偏差
   └── 阈值：>5%触发告警

2. PSI（Population Stability Index）
   └── 特征分布漂移
   └── 阈值：>0.1黄色预警，>0.25红色强制回滚

3. AUC实时监控
   └── 滑动窗口计算
   └── 下降>0.5%触发重训
```

## 5.6 容灾与降级策略

### 多级降级方案

```
降级优先级：

Level 1: 特征降级
├── 实时特征 → 近线特征
└── 复杂特征 → 简单特征

Level 2: 模型降级
├── 大模型 → 小模型
└── 精排 → 粗排结果

Level 3: 召回降级
├── 多路召回 → 单路召回
└── 个性化 → 热门推荐

Level 4: 兜底策略
└── 返回缓存结果或默认推荐
```

### 灰度发布流程

```
灰度发布步骤：

1. 影子流量测试
   └── 1%流量走新模型，只记录不打乱

2. 小流量灰度
   └── 5%流量灰度，观察核心指标

3. 分段放量
   └── 10% → 30% → 50% → 100%
   └── 每阶段观察2小时

4. 回滚机制
   └── 任何阶段指标异常，30秒内回滚
```
