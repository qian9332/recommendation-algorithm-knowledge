# 八、广告竞价机制设计

> 广告系统的核心是竞价机制，需要在平台收入、广告主效用、用户体验之间取得平衡。

---

## 8.1 拍卖机制对比

### GSP、VCG、GFP对比

| 机制 | 平台收入 | 广告主效用 | 真实报价激励 | 计算复杂度 |
|-----|---------|-----------|-------------|-----------|
| **GSP** | 中 | 中 | 否 | 低 |
| **VCG** | 低 | 高 | 是 | 高 |
| **GFP** | 高 | 低 | 否 | 低 |

### GSP（Generalized Second Price）详解

```
出价规则：
├── 广告主i出价b_i
├── 按出价排序
└── 第j位广告主支付第j+1位的出价

支付价格计算：
p_j = b_{j+1} · (CTR_{j+1} / CTR_j)

优势：
├── 计算简单
├── 工业界广泛使用
└── 广告主易于理解

劣势：
├── 不激励真实报价
└── 存在策略性出价空间
```

### VCG（Vickrey-Clarke-Groves）详解

```
核心思想：
广告主支付其对其他广告主造成的外部性损失

支付价格计算：
p_i = Σ_{j≠i} (v_j · (x_j^{-i} - x_j))

其中：
├── v_j：广告主j的真实价值
├── x_j：广告主j在有i时的收益
└── x_j^{-i}：广告主j在没有i时的收益

优势：
├── 激励真实报价（Dominant Strategy）
└── 社会福利最大化

劣势：
├── 计算复杂度高
├── 广告主难以理解
└── 工业界落地少
```

## 8.2 OCPC出价策略

### OCPC核心思想

```
目标：在保证转化成本达标的前提下，最大化转化量

公式：
bid = target_cpa × pCVR × 调整系数

其中：
├── target_cpa：目标转化成本
├── pCVR：预估转化率
└── 调整系数：根据实时消耗/转化动态调整
```

### OCPC两阶段策略

```
第一阶段（数据积累期）：
├── 使用保守出价策略
├── 积累转化数据
├── 建立pCVR模型
└── 持续时间：3-7天

第二阶段（智能出价期）：
├── 使用OCPC出价
├── 实时调整出价系数
├── 监控转化成本
└── 动态优化
```

### 出价校准

```
问题：pCVR预估偏差导致出价不准

解决方案：
1. 概率校准
   └── Platt Scaling / Isotonic Regression
   └── 保证pCVR均值与真实CVR一致

2. 成本控制
   └── 实时监控实际CPA
   └── 当实际CPA偏离目标时，调整出价系数

3. 分层出价
   └── 不同转化类型使用不同出价策略
   └── 如：下单出价高于加购出价
```

## 8.3 oCPM/oCPC对比

| 维度 | oCPM | oCPC |
|-----|------|------|
| 计费方式 | 按展示计费 | 按点击计费 |
| 出价目标 | 转化成本 | 转化成本 |
| 风险承担 | 平台承担点击风险 | 广告主承担点击风险 |
| 适用场景 | 品牌广告、低CTR场景 | 效果广告、高CTR场景 |
| 优化难度 | 较高（需预估CTR+CVR） | 较低（只需预估CVR） |

## 8.4 出价策略优化

### 多目标出价

```
场景：同时优化多个转化目标

公式：
bid = w1 × target_cpa1 × pCVR1 + w2 × target_cpa2 × pCVR2 + ...

权重确定：
├── 广告主指定各目标重要性
├── 系统根据历史数据优化权重
└── 动态调整以平衡各目标
```

### 分时段出价

```
场景：不同时段转化效率不同

策略：
├── 高峰时段（转化率高）：提高出价
├── 低谷时段（转化率低）：降低出价
└── 根据历史转化数据自动调整

实现：
├── 建立时段-转化率模型
├── 预测各时段转化率
└── 动态调整出价系数
```

### 竞价密度感知出价

```
场景：不同竞价环境竞争强度不同

策略：
├── 高竞争环境：适当提高出价
├── 低竞争环境：适当降低出价
└── 在保证曝光的前提下优化成本

实现：
├── 实时估算竞价密度
├── 预测获胜概率
└── 根据获胜概率调整出价
```

## 8.5 广告排序公式

### eCPM排序

```
eCPM = pCTR × pCVR × bid × 1000

排序规则：
├── 按eCPM降序排列
├── Top-N广告获得展示机会
└── 广告主按GSP规则支付
```

### 质量分引入

```
问题：低质量广告通过高出价获得曝光

解决方案：引入质量分

排序公式：
rank_score = pCTR × pCVR × bid × quality_score

质量分计算：
├── 落地页质量
├── 广告创意质量
├── 用户反馈（负反馈率）
└── 历史违规记录
```

### 多样性约束

```
问题：同一广告主广告扎堆

解决方案：
├── 同广告主广告打散
├── 限制同广告主广告数量
└── 引入多样性惩罚项

排序公式：
rank_score = eCPM - λ × diversity_penalty

其中：
├── λ：多样性权重
└── diversity_penalty：与已选广告的相似度
```

## 8.6 预算控制

### 预算分配策略

```
日预算分配：
├── 均匀分配：每小时预算 = 日预算 / 24
├── 效率优先：高转化时段多分配
└── 智能分配：根据实时效果动态调整

实现：
├── Pacing算法控制消耗速度
├── 实时监控预算消耗
└── 动态调整出价或暂停投放
```

### Pacing算法

```
目标：在预算约束下最大化转化量

算法：
├── 计算当前消耗速度 vs 目标消耗速度
├── 若消耗过快：降低出价或减少投放
├── 若消耗过慢：提高出价或增加投放
└── 平滑调整，避免剧烈波动

参数：
├── 调整周期：5-15分钟
├── 调整幅度：5-20%
└── 紧急刹车阈值：预算消耗>120%
```

## 8.7 广告效果归因

### 归因模型对比

| 模型 | 说明 | 适用场景 |
|-----|------|---------|
| 最后点击归因 | 转化归因于最后一次点击 | 效果广告 |
| 首次点击归因 | 转化归因于第一次点击 | 品牌广告 |
| 线性归因 | 转化均分给所有触点 | 多触点场景 |
| 时间衰减归因 | 近期触点权重更高 | 长转化周期 |
| 数据驱动归因 | 基于模型计算各触点贡献 | 大数据场景 |

### 归因窗口

```
常见归因窗口：
├── 点击归因窗口：7-30天
├── 曝光归因窗口：1-7天
└── 根据行业特点调整

跨渠道归因：
├── 整合多渠道数据
├── 统一用户标识
└── 计算各渠道贡献
```

## 8.8 广告反作弊

### 常见作弊类型

| 类型 | 特征 | 检测方法 |
|-----|------|---------|
| 点击欺诈 | 短时间大量点击 | IP/设备频次分析 |
| 虚假转化 | 异常转化模式 | 转化时间分布分析 |
| 竞品恶意点击 | 针对性消耗预算 | 行为模式识别 |
| 机器流量 | 非人类行为 | 行为特征分析 |

### 反作弊策略

```
实时检测：
├── IP黑名单
├── 设备指纹识别
├── 行为序列分析
└── 异常流量过滤

离线分析：
├── 历史数据挖掘
├── 作弊模式学习
├── 模型训练
└── 规则更新

补偿机制：
├── 异常点击不计费
├── 虚假转化退款
└── 预算补偿
```
