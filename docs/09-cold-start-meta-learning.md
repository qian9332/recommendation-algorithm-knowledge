# 九、推荐冷启动与元学习

> 冷启动是推荐系统的经典难题，涉及新用户、新商品、新场景等多种场景。

---

## 9.1 冷启动场景分类

### 三类冷启动

| 类型 | 挑战 | 解决思路 |
|-----|------|---------|
| **新用户冷启动** | 无历史行为 | 人口统计学特征、注册信息、实时行为 |
| **新商品冷启动** | 无交互数据 | 内容特征、知识图谱、迁移学习 |
| **新场景冷启动** | 无场景数据 | 跨场景迁移、元学习 |

### 冷启动评估指标

| 指标 | 说明 | 目标值 |
|-----|------|--------|
| 首日CTR | 新商品首日点击率 | >基准80% |
| 首周GMV | 新商品首周成交额 | >基准70% |
| 冷启动覆盖率 | 新商品获得曝光的比例 | >90% |
| 长尾商品曝光占比 | 长尾商品曝光量占比 | >30% |

## 9.2 新商品冷启动

### 零样本向量构建

```
输入特征：
├── 商品类目ID
├── 商品属性（品牌、价格带、规格）
├── 商品标题（BERT编码）
├── 商品主图（CNN编码）
└── 卖家特征（店铺等级、历史销量）

融合策略：
├── 特征拼接 + MLP
├── 注意力融合
└── 图神经网络聚合

输出：
└── 商品冷启动向量（128维）
```

### 图文多模态预训练

```
架构：
├── 图像编码器：ResNet-50
├── 文本编码器：BERT-base
├── 跨模态对齐：CLIP-style对比学习
└── 融合层：Cross-attention

训练数据：
├── 商品主图 + 标题对
├── 亿级规模
└── 自监督学习

效果：
├── 新商品CTR提升：+15%
└── 零样本向量质量显著提升
```

### 知识图谱辅助

```
构建商品知识图谱：
├── 节点：商品、类目、品牌、属性
├── 边：属于、相似、替代、互补
└── 权重：关系强度

冷启动利用：
├── 新商品连接到图谱
├── 聚合邻居节点信息
├── GraphSAGE / GAT编码
└── 得到商品表示

优势：
├── 利用结构化知识
├── 可解释性强
└── 泛化能力好
```

## 9.3 新用户冷启动

### 人口统计学特征

```
可用特征：
├── 年龄、性别、地域
├── 设备类型、操作系统
├── 注册渠道、注册时间
└── 人口统计学标签

建模方法：
├── 特征Embedding
├── 与商品特征交叉
└── 决策树规则补充

效果：
├── 新用户首日CTR提升：+10%
└── 快速建立初始画像
```

### 实时行为序列

```
场景：用户刚进入App，产生少量行为

利用方式：
├── 实时捕获用户行为（点击、浏览、搜索）
├── 构建短期行为序列
├── Transformer编码
└── 生成实时用户向量

技术要点：
├── 行为序列长度：5-20
├── 时间窗口：最近30分钟
├── 行为类型加权：点击>浏览
└── 实时更新频率：秒级
```

### 跨域迁移

```
场景：用户在其他域有丰富行为

迁移策略：
├── 源域：用户行为丰富的域（如搜索）
├── 目标域：用户行为稀疏的域（如推荐）
├── 共享用户表示
└── 域适应技术

实现：
├── 双塔结构，共享用户塔
├── 域判别器 + 对抗训练
└── 渐进式迁移

效果：
├── 新用户CTR提升：+20%
└── 加速用户画像建立
```

## 9.4 元学习（Meta-Learning）

### MAML框架

```
目标：学习一个好的初始化参数，使模型能快速适应新任务

二阶梯度更新：
θ' = θ - α∇L_train(θ)
θ_new = θ - β∇L_val(θ')

其中：
├── θ：模型参数
├── α：内层学习率
├── β：外层学习率
├── L_train：训练集损失
└── L_val：验证集损失
```

### MAML在冷启动中的应用

```
任务定义：
├── 每个用户/商品是一个任务
├── 支持集：少量交互数据
└── 查询集：评估泛化能力

训练流程：
├── 采样一批任务
├── 内层更新：用支持集微调参数
├── 外层更新：用查询集优化初始化
└── 迭代训练

效果：
├── 新用户冷启动：CTR提升+15%
├── 新商品冷启动：CTR提升+12%
└── 快速适应：3-5次交互即可收敛
```

### 元学习变体对比

| 方法 | 计算复杂度 | 效果 | 适用场景 |
|-----|-----------|------|---------|
| MAML | 高（二阶导） | 好 | 通用 |
| First-order MAML | 中 | 中 | 计算资源受限 |
| Reptile | 低 | 中 | 快速原型 |
| Meta-SGD | 高 | 好 | 需要学习学习率 |

## 9.5 探索与利用

### 探索策略

```
目标：给新商品/新用户足够的曝光机会

策略：
1. ε-greedy
   └── 以ε概率随机选择新商品
   └── ε通常取0.05-0.1

2. UCB（Upper Confidence Bound）
   └── score = μ + c·√(ln(N)/n)
   └── 平衡均值和不确定性

3. Thompson Sampling
   └── 从后验分布采样
   └── 贝叶斯方法

4. 个性化探索
   └── 根据用户特征调整探索强度
   └── 新用户探索强度更高
```

### 探索预算分配

```
业务约束：
├── 总探索流量占比：5-10%
├── 新商品最低曝光量：100次
└── 探索期间GMV损失：<3%

分配策略：
├── 按商品潜力分配预算
├── 实时监控探索效果
├── 动态调整预算分配
└── 及时止损/加码
```

## 9.6 冷启动效果评估

### 离线评估

```
评估方法：
├── 时间切分：用历史数据模拟冷启动
├── 留一评估：隐藏部分交互数据
└── 少样本学习：只用少量样本训练

指标：
├── Hit Rate@K：新商品被召回的比例
├── NDCG@K：排序质量
└── AUC：分类能力
```

### 在线评估

```
A/B实验设计：
├── 实验组：冷启动策略
├── 对照组：基线策略
├── 流量分配：5%
└── 实验周期：7-14天

核心指标：
├── 新商品首日CTR
├── 新商品首周GMV
├── 新用户留存率
└── 整体GMV影响
```

## 9.7 冷启动与长期价值

### 冷启动与用户留存

```
问题：过度探索可能损害用户体验

平衡策略：
├── 新用户：探索强度高，但保证相关性
├── 老用户：探索强度低，保证精准度
├── 实时监控用户反馈
└── 动态调整探索策略

长期价值建模：
├── 引入CLV（客户终身价值）
├── 冷启动决策考虑长期影响
└── 强化学习优化长期收益
```

### 冷启动与生态健康

```
生态视角：
├── 新商品需要曝光机会
├── 新商家需要成长空间
├── 用户需要新鲜感
└── 平台需要多样性

健康指标：
├── 新商品存活率
├── 新商家留存率
├── 商品基尼系数
└── 用户满意度
```
