# 七、粗排模型蒸馏与加速

> 粗排是召回与精排之间的关键环节，需要在极低延迟（<5ms）下完成大规模候选集的初步筛选。

---

## 7.1 知识蒸馏框架设计

### List-wise软标签构造

```
核心思想：用精排模型对候选集打分，保留排序关系

步骤：
1. 精排模型对候选集C中的每个商品打分
2. 对分数做温度缩放：p_i = exp(s_i/τ) / Σ exp(s_j/τ)
3. 温度系数τ控制软标签平滑度
   └── τ越大，分布越平滑
   └── τ越小，分布越尖锐

经验值：τ=2~5效果最佳
```

### 蒸馏Loss设计

```python
# 蒸馏损失公式
L_distill = KL(P_teacher || P_student)

# 总损失
L_total = α·L_distill + (1-α)·L_hard

# 其中：
# - L_distill: 蒸馏损失（软标签）
# - L_hard: 硬标签交叉熵
# - α: 蒸馏权重，通常取0.5~0.8
```

### 蒸馏效果评估

| 指标 | 计算方式 | 目标值 |
|-----|---------|--------|
| Kendall Tau | 粗排与精排Top-200结果一致性 | >0.85 |
| Recall@K | 精排Top-K被粗排保留的比例 | >95% |
| Latency | 粗排P99延迟 | <5ms |

## 7.2 低延迟模型结构选择

### 候选模型对比

| 模型 | 5ms延迟下效果 | 参数量 | 特点 |
|-----|-------------|--------|------|
| **DCN-M** | 中 | 小 | 显式特征交叉 |
| **FiBiNET** | 中高 | 中 | SENet特征重要性学习 |
| **SimpleDNN** | 高 | 小 | 结构简单，延迟低 |
| **LightGBM** | 高 | - | 树模型，CPU友好 |

### 推荐方案

```
场景1：GPU推理
├── 推荐SimpleDNN或轻量DCN-M
├── 特征交叉控制在2层以内
└── 延迟可控在3ms以内

场景2：CPU推理
├── 推荐LightGBM（40棵树，深度6）
├── 单次推理<1ms
└── 无需GPU资源
```

## 7.3 特征穿越问题

### 问题定义

```
特征穿越：训练时使用了线上推理时不可用的特征

常见场景：
1. 使用了"点击后"才能获得的特征（如停留时长）
2. 使用了"未来"的特征（如商品后续销量）
3. 使用了"位置"特征（推理时位置未知）
```

### 检测与修复

```
检测方法：
1. 特征时间戳校验
   └── 确保所有特征时间戳 < 样本时间戳

2. 特征可用性监控
   └── 线上实时统计特征缺失率

修复方案：
1. 特征快照
   └── 训练时使用特征快照，而非实时特征

2. 特征对齐
   └── 确保训练和推理使用相同的特征计算逻辑

3. 延迟特征处理
   └── 对延迟到达的特征做特殊处理（如填充默认值）
```

## 7.4 粗排与精排一致性优化

### 一致性问题

```
问题：粗排筛选后的候选集，精排可能给出完全不同的排序

原因：
1. 粗排和精排使用不同的特征集
2. 粗排和精排模型结构差异大
3. 粗排模型未充分学习精排的排序偏好
```

### 优化方案

```
方案1：特征对齐
├── 粗排使用精排特征的子集
├── 确保核心特征一致
└── 减少特征差异带来的排序偏差

方案2：蒸馏对齐
├── 用精排输出作为粗排的训练目标
├── 不仅学习点击标签，还学习精排分数
└── 提升粗排与精排的排序一致性

方案3：联合训练
├── 粗排和精排共享部分参数
├── 通过多任务学习提升一致性
└── 需要精心设计训练流程
```

## 7.5 粗排候选集规模优化

### 候选集规模权衡

| 候选集规模 | 召回覆盖率 | 粗排延迟 | 精排负载 |
|-----------|-----------|---------|---------|
| 500 | 低 | 低 | 低 |
| 1000 | 中 | 中 | 中 |
| 2000 | 高 | 高 | 高 |
| 5000 | 最高 | 最高 | 最高 |

### 动态候选集规模

```
根据请求复杂度动态调整候选集规模：

简单请求（用户历史少、Query明确）：
└── 候选集规模=500

复杂请求（用户历史多、Query模糊）：
└── 候选集规模=2000

实现：
├── 根据用户特征复杂度评分
├── 映射到候选集规模档位
└── 在线动态调整
```

## 7.6 粗排模型压缩

### 压缩技术对比

| 技术 | 压缩比 | 精度损失 | 延迟降低 |
|-----|--------|---------|---------|
| 知识蒸馏 | 10× | <1% | 50% |
| INT8量化 | 4× | <0.5% | 30% |
| 剪枝 | 2-5× | <1% | 20-40% |
| 算子融合 | - | 0% | 15% |

### 推荐压缩流程

```
Step 1: 知识蒸馏
├── 用精排模型作为教师
├── 训练小型学生模型
└── 压缩比：10×

Step 2: INT8量化
├── 对学生模型做INT8量化
├── 使用校准数据集
└── 精度损失：<0.5%

Step 3: 算子融合
├── 使用TensorRT或ONNXRuntime
├── 融合相邻算子
└── 延迟降低：15%

最终效果：
├── 模型大小：从100MB降到2.5MB
├── P99延迟：从15ms降到3ms
└── AUC损失：<0.3%
```

## 7.7 粗排在线学习

### 在线学习必要性

```
离线训练的问题：
1. 数据分布漂移（概念漂移）
2. 新商品冷启动
3. 大促期间行为模式突变

在线学习的优势：
1. 实时适应分布变化
2. 快速响应新趋势
3. 提升模型时效性
```

### 在线学习实现

```
架构：
├── Flink实时特征计算
├── Parameter Server增量更新
└── 模型热加载

关键参数：
├── 学习率：1e-5 ~ 1e-4（比离线小）
├── 批次大小：1000 ~ 5000
├── 更新频率：每分钟
└── 梯度裁剪：防止异常样本破坏模型

监控指标：
├── 在线AUC
├── 模型参数漂移
└── 预测分布稳定性
```
