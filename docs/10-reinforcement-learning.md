# 十、强化学习在推荐中的应用

> 强化学习为推荐系统提供了优化长期收益的框架，是近年来工业界研究的热点方向。

---

## 10.1 推荐系统中的MDP建模

### MDP五元组定义

| 要素 | 推荐场景定义 |
|-----|-------------|
| **状态S** | 用户历史行为序列、用户画像、上下文特征 |
| **动作A** | 推荐的商品列表/单个商品 |
| **奖励R** | 点击、转化、停留时长、GMV等加权 |
| **转移P** | 用户对推荐的反馈，决定下一状态 |
| **折扣γ** | 平衡短期与长期收益，通常取0.9-0.99 |

### 状态设计

```
状态向量组成：
├── 用户侧
│   ├── 近N次行为序列（Transformer编码）
│   ├── 长期兴趣向量
│   └── 实时上下文（时间、位置、设备）
├── 商品侧
│   ├── 候选商品特征
│   └── 候选商品与用户兴趣的匹配度
└── 历史交互
    ├── 已推荐商品列表
    └── 用户反馈序列

状态维度：通常128-512维
```

### 动作设计

```
动作空间选择：

1. 离散动作空间
   ├── 动作 = 从候选集中选择一个商品
   ├── 动作空间大小 = 候选集大小
   └── 适用：小规模候选集

2. 连续动作空间
   ├── 动作 = 商品嵌入向量
   ├── 通过最近邻检索商品
   └── 适用：大规模候选集

3. 序列动作空间
   ├── 动作 = 商品排列顺序
   ├── Pointer Network输出
   └── 适用：重排场景
```

### 奖励设计

```
奖励函数设计原则：
├── 可观测：奖励信号能及时获得
├── 可量化：奖励能数值化
├── 与目标对齐：奖励反映业务目标
└── 稀疏与密集平衡：避免纯稀疏奖励

典型奖励函数：
R = α·点击 + β·转化 + γ·停留时长 - δ·负反馈

权重调整：
├── 根据业务阶段调整
├── 大促期间提高转化权重
└── 冷启动阶段提高探索奖励
```

## 10.2 策略梯度方法

### REINFORCE

```
策略梯度公式：
∇J(θ) = E[∇logπ(a|s) · R]

优点：
├── 无需价值函数
├── 理论简单
└── 适用于离散动作

缺点：
├── 方差大
├── 样本效率低
└── 收敛慢

工业界改进：
├── 引入基线（baseline）降低方差
├── 使用因果基线
└── 结合重要性采样
```

### Actor-Critic

```
架构：
├── Actor：策略网络，输出动作概率
├── Critic：价值网络，估计状态价值
└── 联合训练

优势函数：
A(s,a) = Q(s,a) - V(s)

更新公式：
├── Actor: ∇θ ≈ ∇logπ(a|s) · A(s,a)
└── Critic: 最小化TD误差

工业界实践：
├── 方差比REINFORCE降低4倍
├── 成为美团、抖音、淘宝主流方案
└── 配合离线预训练效果更佳
```

### PPO（Proximal Policy Optimization）

```
核心思想：限制策略更新幅度，保证稳定性

目标函数：
L(θ) = E[min(r(θ)·A, clip(r(θ), 1-ε, 1+ε)·A)]

其中：
├── r(θ) = π_θ(a|s) / π_θ_old(a|s)
├── A：优势函数
└── ε：裁剪参数，通常取0.1-0.2

优点：
├── 训练稳定
├── 超参数少
└── 工业界广泛使用
```

## 10.3 Off-policy学习

### 问题与挑战

```
挑战：
├── 训练数据来自历史策略（行为策略）
├── 目标是优化新策略（目标策略）
├── 分布不匹配导致偏差
└── 重要性权重方差大

解决方案：
├── 重要性采样（IPS）
├── 双重稳健估计（DR）
├── 值函数学习（FQI）
└── 约束策略优化
```

### 重要性采样（IPS）

```
IPS估计：
V̂_IPS = (1/n) Σ (π(a|s) / β(a|s)) · r

其中：
├── π：目标策略
├── β：行为策略
└── r：奖励

问题：
├── 权重可能极大
├── 方差爆炸
└── 需要裁剪

改进：
├── 权重裁剪
├── 自归一化IPS（SNIPS）
└── 双重稳健（DR）
```

### 双重稳健估计（DR）

```
DR估计：
V̂_DR = (1/n) Σ [ρ·(r - Q̂) + E_π[Q̂]]

其中：
├── ρ = π/β：重要性权重
├── Q̂：价值函数估计
└── E_π[Q̂]：策略期望价值

优点：
├── 当Q̂准确时，方差低
├── 当Q̂不准时，退化为IPS，仍无偏
└── 双重稳健

工业界效果：
├── 方差降低60-70%
└── 成为Off-policy评估标配
```

## 10.4 探索策略

### ε-greedy

```
策略：
├── 以ε概率随机探索
├── 以1-ε概率利用当前最优
└── ε通常从0.1逐渐衰减

优点：
├── 实现简单
├── 易于控制探索强度
└── 工业界广泛使用

缺点：
├── 探索效率低
├── 随机探索可能推荐不相关商品
└── 不考虑不确定性
```

### UCB（Upper Confidence Bound）

```
策略：
选择 a* = argmax [μ(a) + c·√(ln(N)/n(a))]

其中：
├── μ(a)：动作a的平均奖励
├── N：总尝试次数
├── n(a)：动作a的尝试次数
└── c：探索系数

优点：
├── 理论最优遗憾界
├── 自动平衡探索与利用
└── 考虑不确定性

缺点：
├── 需要维护每个动作的统计量
├── 动作空间大时难以应用
└── 需要离散动作空间
```

### Thompson Sampling

```
策略：
├── 维护奖励分布的后验
├── 从后验采样参数
├── 用采样参数计算最优动作
└── 执行并更新后验

优点：
├── 贝叶斯最优
├── 自然平衡探索与利用
└── 适用于各种奖励分布

缺点：
├── 需要指定先验
├── 后验更新可能复杂
└── 计算开销较大
```

## 10.5 多目标强化学习

### 向量奖励

```
场景：同时优化多个目标

奖励向量：
R = [R_CTR, R_CVR, R_时长, R_多样性]

策略优化：
├── Pareto最优
├── 加权标量化
└── 约束优化

加权标量化：
R_total = w1·R_CTR + w2·R_CVR + w3·R_时长 + w4·R_多样性

权重调整：
├── 业务阶段不同，权重不同
├── 大促期提高GMV权重
└── 日常期提高留存权重
```

### 约束强化学习

```
场景：在满足约束的前提下优化目标

形式化：
max E[Σ γ^t R_t]
s.t. E[Σ γ^t C_t] ≤ 阈值

其中：
├── R_t：奖励
├── C_t：约束成本
└── 如：CTR最大化，但负反馈率≤阈值

算法：
├── Lagrangian方法
├── CPO（Constrained Policy Optimization）
└── 安全强化学习
```

## 10.6 强化学习落地挑战

### 延迟奖励

```
问题：
├── 转化可能延迟数小时甚至数天
├── 训练时奖励未知
└── 影响策略学习

解决方案：
├── 奖励插补：用模型预估延迟奖励
├── 中间奖励：设计即时反馈信号
├── 延迟奖励建模：用生存分析
└── 多步TD学习：λ-return
```

### 样本效率

```
问题：
├── 强化学习样本效率低
├── 需要大量交互数据
└── 工业界数据宝贵

解决方案：
├── Off-policy学习：利用历史数据
├── 模仿学习：先学专家策略
├── 模型-based RL：学习环境模型
└── 离线预训练 + 在线微调
```

### 安全性与可解释性

```
安全性：
├── 防止策略崩溃
├── 设置安全约束
├── 实时监控异常
└── 快速回滚机制

可解释性：
├── 策略决策可解释
├── 奖励归因分析
├── 用户反馈可视化
└── 满足监管要求
```

## 10.7 强化学习在推荐中的典型应用

### 重排场景

```
场景特点：
├── 候选集固定（50-200个商品）
├── 需要考虑商品间相互影响
├── 目标是整体列表收益最大化
└── 天然适合序列决策

方法：
├── Pointer Network输出排列
├── List-wise奖励
├── Actor-Critic训练
└── 效果：GMV提升3-5%
```

### 长期价值优化

```
场景特点：
├── 短期CTR优化可能损害长期留存
├── 需要平衡短期与长期收益
└── 强化学习天然适合

方法：
├── 奖励包含长期价值（CLV）
├── 折扣因子γ控制长期权重
├── 状态包含用户生命周期阶段
└── 效果：留存率提升2-3%
```

### 探索策略优化

```
场景特点：
├── 需要给新商品/新用户曝光机会
├── 探索可能损害短期收益
└── 需要智能探索

方法：
├── UCB/Thompson Sampling
├── 不确定性估计
├── 个性化探索强度
└── 效果：新商品冷启动速度提升40%
```
